# Contest IO

[![Go Version](https://img.shields.io/github/go-mod/go-version/aaa2ppp/contestio)](go.mod)
[![License](https://img.shields.io/github/license/aaa2ppp/contestio)](LICENSE)

**Contest IO** — библиотека для быстрого ввода/вывода в алгоритмическом программировании на Go.  

> **Статус:** Бета-версия (v0.1). Основное внимание уделяется стабильности, однако отдельные недочёты пока не исключены. Любая обратная связь (issues, PR) помогает улучшать библиотеку.

## Почему Contest IO?
- **Скорость разработки** — чтение и запись массивов одной строкой, без циклов и шаблонного кода. Вы пишете только логику решения.
- **Скорость выполнения** — производительность, максимально близкая к ручному коду.
- **Нулевые аллокации** — память расходуется только на ваши данные. Вы платите только за то, что действительно используете.


## Мотивация

В соревнованиях по программированию (Codeforces, Яндекс.Контест и др.) каждая минута на счету.  
Писать ввод-вывод вручную — это:

*   **Риск ошибок** — легко перепутать индекс, забыть `&`, не обработать конец строки.
*   **Потеря времени** — отладка таких мест отнимает время, которое можно было потратить на алгоритм.
*   **Снижение читаемости** — код решения тонет в служебных конструкциях.

В C++ всё просто: `cin >> n; for(...) cin >> a[i];`. В Go так не получается.  
Писать свой парсер с буферами и токенами в стрессовой ситуации соревнования — верный способ наделать ошибок и потерять драгоценное время.
А стандартный `fmt` на больших входных данных (миллионы чисел) превращается в узкое место: рефлексия, аллокации на куче, необходимость писать циклы...

`contestio` решает эти проблемы! Она даёт вам скорость ручного кода и простоту `fmt`.

## Ключевые особенности

- **Эффективный токенизатор**  
  Разбирает входной поток на лексемы быстрее, чем `bufio.Scanner`, и без единой аллокации.

- **Быстрый парсер целых чисел**  
  Быстрее `strconv.Atoi` в 1.5 раза и поддерживает все целые типы (`int`, `int8`…`int64`, `uint`…`uint64`) с полной проверкой переполнения — без потери скорости.

- **Нулевые аллокации**  
  Парсинг и вывод не выделяют память в куче (если вы заранее выделили слайс). Сборщик мусора отдыхает.

- **Обобщённость через дженерики**  
  Одни и те же функции работают с `int`, `int32`, `uint64`, `float32`, `float64` — никакого копипаста.

- **Гибкость**  
  Читайте фиксированное количество чисел, все числа из строки, отдельные значения — как удобно.

- **Встраивание кода одной командой**  
  Утилита `contestio-inline` добавляет в ваше решение необходимые функции библиотеки — идеально для систем, запрещающих внешние зависимости.
  
- **Совместимость**  
  `Reader` и `Writer` библиотеки — это минимальные обертки над `bufio.Reader/Writer`. 
  Вам доступны все методы `bufio`, а при необходимости вы можете использовать `fmt.Fscan/Fprint`.

## Быстрый старт

### Установка

```sh
go get github.com/aaa2ppp/contestio@latest
```

### Пример: читаем массив, обрабатываем, выводим

```go
package main

import (
    "os"
    . "github.com/aaa2ppp/contestio" // точечный импорт для краткости
)

func main() {
    br := NewReader(os.Stdin)
    bw := NewWriter(os.Stdout)
    defer bw.Flush()

    // Читаем размер массива
    var n int
    if _, err := ScanIntLn(br, &n); err != nil {
        panic(err) // В задачах ввод всегда корректен, поэтому можно падать
    }

    // Читаем сам массив (одна строчка!)
    a := make([]int, n)
    if _, err := ScanInts(br, a); err != nil {
        panic(err)
    }

    // Ваше решение
    for i := range a {
        a[i] *= a[i]
    }

    // Выводим результат (тоже одна строчка)
    PrintIntsLn(bw, a)
}
```

### Чтение всех чисел из строки (неизвестная длина)

```go
a, err := ScanIntsLn(br, nil) // nil — создаст новый слайс
```

### Вывод с разделителями

```go
op := WO{Begin: "[", Sep: ", ", End: "]\n"}
PrintInts(bw, op, []int32{1, 2, 3, 4}) // [1, 2, 3, 4]
```

### Чтение матрицы
```go
var rows, cols int
ScanIntLn(br, &rows, &cols) // можно читать несколько переменных сразу

matrix := make([][]int, rows)
for i := range matrix {
    matrix[i] = make([]int, cols)
    ScanInts(br, matrix[i]) // читаем строку матрицы
}
```

### Смешанный ввод (числа и строки)
```go
var count int
var name string
ScanIntLn(br, &count)          // читаем число
name, _ = br.ReadString('\n')  // стандартный bufio.Reader для строки
name = strings.TrimSpace(name)
```

## Утилита `contestio-inline`:

Многие проверяющие системы (Яндекс.Контест, Codeforces) запрещают внешние зависимости.  
Можно держать шаблоны со всеми парсерами и копировать их в каждое решение — громоздко и неудобно.

**`contestio-inline`** элегантно решает эту проблему:

- Вы пишете и тестируете решение, используя точечный импорт `contestio` — код остаётся чистым и кратким.
- Перед отправкой выполняете одну команду — и утилита встраивает в ваш файл только те функции библиотеки, которые реально используются.
- Полученный файл самодостаточен и готов к отправке на проверку.

```sh
# Установка
go install github.com/aaa2ppp/contestio/cmd/contestio-inline@latest

# Инлайн (работает только с точечным импортом!)
contestio-inline main.go

# Файл main.go теперь содержит весь необходимый код.
# Для отката: contestio-inline -clear main.go
```

**Важно:** Утилита требует точечный импорт (`import . "github.com/aaa2ppp/contestio"`). Обычный или алиасный импорт не поддерживается.

## Производительность

Все бенчмарки — для **1 000 000 чисел** (`int` или `float64`), буфер 4 КБ.  
**Система:** Intel Core i3-7100 @ 3.90GHz, Windows, Go 1.24.2.

В таблицах **жирным** выделены функции `contestio`.  
Время в миллисекундах — чем меньше, тем лучше.

## Теги сборки: выбор парсера целых чисел

По умолчанию используется базовая реализация (`base`) — почти не уступает `fast` по скорости, но содержит меньше кода.  
Если нужно выжать максимум, используйте тег `parse_int_fast`:

```sh
go build -tags=parse_int_fast
```

Доступные теги (взаимоисключающие):
- `parse_int_fast` — максимальная производительность для десятичных целых.
- `parse_int_std` — использовать `strconv.ParseInt`/`ParseUint` (максимальная надёжность).

## Расширенное использование: API с интерфейсами (тег `sugar`)

Для тех, кто предпочитает большего обобщения, в библиотеке предусмотрен опциональный «сахарный» слой. 
Он активируется тегом сборки `-tags=sugar` и добавляет типы-обёртки над слайсами (например, `Ints[T]`) реализующие интерфейсы,
которые используются универсальными функциями `ScanSlice` и `PrintSlice`.

**Пример использования сахара:**

```go
package main

import (
    "os"
    . "github.com/aaa2ppp/contestio"
)

func solve(a []int) []int {
    for i := range a {
        a[i] *= a[i]
    }
    return a
}

func main() {
    br := NewReader(os.Stdin)
    bw := NewWriter(os.Stdout)
    defer bw.Flush()

    var n int
    var a Ints[int] // Используем сахарный тип Ints[int]

    if _, err := ScanIntLn(br, &n); err != nil {
        panic(err)
    }
    
    a = Resize(a, n)
    if _, err := ScanSlice(br, a); err != nil { // Читаем через общий интерфейс
        panic(err)
    }

    a = solve(a)

    PrintSliceLn(bw, a) // Вывод через общий интерфейс
}
```

**Особенности:**
- Позволяет писать более абстрактный код, не привязываясь к конкретным типам функций (`ScanInts`, `PrintInts` и т.д.).
- Использует интерфейсные обертки над срезами. Их можно прозрачно передавать, принимать и работать, как с обычными срезами.
- Требует указания тега сборки при компиляции: `go build -tags=sugar`.
- Для комфортной работы в VS Code необходимо добавить `"go.buildTags": "sugar"` в настройки проекта (иначе автодополнение и тесты не увидят код с тегом).

**Важно:** Сахар **не обязателен** для эффективного использования библиотеки. Основной API (`ScanInts`, `PrintInts` и т.п.) полностью покрывает все нужды участников соревнований и не требует дополнительных тегов. Сахар — это дополнительный слой для тех, кто хочет экспериментировать с интерфейсами или хочет использовать библиотеку вне контекста соревнований.

## Настройка VS Code для работы с тегом `sugar`

Если вы используете «сахарный» API, добавьте в `.vscode/settings.json`:

```json
"go.buildTags": "sugar"
```

После этого автодополнение и тесты будут видеть код с тегом.
