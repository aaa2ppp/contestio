# Contest IO

**Contest IO** — это библиотека для быстрого ввода/вывода в алгоритмическом программировании на Go. Она представляет собой набор функций и обёрток над `bufio.Reader` и `bufio.Writer`, которые позволяют читать и выводить данные с производительностью, близкой к ручному коду, и с нулевыми аллокациями.

## Мотивация

В соревнованиях по программированию (Codeforces, Яндекс.Контест и др.) стандартный пакет `fmt` часто становится узким местом. Он удобен, но его цена высока:

*   **Рефлексия и аллокации:** `fmt.Fscan*` и `fmt.Fprintf*` используют рефлексию и создают множество временных объектов, что приводит к огромному количеству аллокаций и замедляет работу на больших объёмах данных (миллионы чисел).
*   **Циклы:** Чтение массива известной длины требует цикла с отдельным вызовом `fmt.Fscan`, что добавляет накладные расходы на каждую итерацию.
*   **Несовместимость:** `bufio.Scanner` отлично справляется с разбиением на токены без аллокаций, но его API не позволяет легко читать и числа, и строки вперемешку.

`contestio` решает эти проблемы, предлагая единый, быстрый и экономичный интерфейс для ввода и вывода.

## Ключевые особенности

*   **Скорость и эффективность:** Работает в разы быстрее `fmt` на больших объёмах данных.
*   **Нулевые аллокации:** Парсинг чисел и вывод происходят без выделения памяти в куче (если слайсы для результатов заранее выделены). Библиотека не создает лишних строк и не нагружает сборщик мусора.
*   **Обобщённость:** Единые функции для всех целых (`int`, `int32`, `uint64` и т.д.) и чисел с плавающей точкой (`float32`, `float64`) через дженерики Go.
*   **Гибкость:** Поддерживаются разные сценарии — чтение фиксированного количества чисел, чтение всех чисел из строки, чтение одиночных значений.
*   **Компактность:** Утилита `contestio-inline` позволяет встроить только необходимый код библиотеки в ваш файл решения, что критично для систем проверки, запрещающих внешние зависимости.

## Быстрый старт

### 1. Установка

```sh
go get github.com/aaa2ppp/contestio
```

### 2. Базовый пример

```go
package main

import (
    "os"
    . "github.com/aaa2ppp/contestio" // Точечный импорт
)

func main() {
    br := NewReader(os.Stdin)   // Создаём читателя
    bw := NewWriter(os.Stdout)  // Создаём писателя
    defer bw.Flush()            // Сбрасываем буфер в конце

    // Читаем первое число (размер массива)
    var n int
    if _, err := ScanIntLn(br, &n); err != nil {
        // В задачах ввод всегда корректен, поэтому можно падать
        panic(err)
    }

    // Читаем массив из n чисел
    a := make([]int, n)
    if _, err := ScanInts(br, a); err != nil {
        panic(err)
    }

    // Ваше решение (пример)
    for i := range a {
        a[i] *= a[i]
    }

    // Выводим результат
    PrintIntsLn(bw, a)
}
```

### 3. Чтение массива неизвестной длины

```go
// Читаем все числа из текущей строки до её конца
a, err := ScanIntsLn(br, nil) // Если передан nil, создаст новый слайс
```

### 4. Использование «сахара» (с тегом `sugar`)

Для более унифицированного кода можно использовать "сахарные" обобщённые обёртки над срезами (например, `Ints[T]`),
которые активируются тегом сборки `-tags=sugar`. Они добавляют интерфейсные типы и позволяют применять
универсальные функции `ScanSlice` и `PrintSlice` вместо набора типизированных функций.
Их можно передавать, принимать и работать, как с обычными срезами.

```go
package main

import (
    "os"
    . "github.com/aaa2ppp/contestio"
)

func solve(a []int) []int {
	for i := range a {
		a[i] = a[i] * a[i]
	}
	return a
}

func main() {
    br := NewReader(os.Stdin)
    bw := NewWriter(os.Stdout)
    defer bw.Flush()

    var n int
    if _, err := ScanIntLn(br, &n); err != nil {
        panic(err)
    }

    var a Ints[int] // Сахарный тип
    a = Resize(a, n)
    if _, err := ScanSlice(br, a); err != nil { // Читаем через общий интерфейс
        panic(err)
    }

    a = solve(a)

    PrintSliceLn(bw, a) // Выводим через общий интерфейс
}
```
**Важно:** Файлы с тегом `sugar` не компилируются по умолчанию. При использовани go `build`, `run`, `test`... нужно указывать
указывать тег сборки `-tags=sugar`. Чтобы они подхватывались инструментарием (автодополнение, тесты), нужно настроить IDE.

#### Настройка VS Code для автоматического использования тега `sugar`

Добавьте в файл `.vscode/settings.json` вашего проекта (или в глобальные настройки) строку:

```json
"go.buildTags": "sugar"
```

После этого все инструменты Go (включая `gopls`, линтер, тестовый раннер) будут использовать тег `sugar` при анализе и запуске тестов.

### 5. Инлайн кода перед отправкой (`contestio-inline`)

Утилита `contestio-inline` встраивает код библиотеки прямо в ваш файл решения, удаляя импорт и делая его самодостаточным.

```sh
# Установка
go install github.com/aaa2ppp/contestio/cmd/contestio-inline@latest

# Переход в директорию с решением и инлайн
cd your_solution
contestio-inline main.go

# Файл main.go готов к отправке на проверку.
# Для отката изменений используйте -clear
# contestio-inline -clear main.go
```

Утилита поддерживат флаг тегов сборки `-tags`.

**Важно:** Утилита `contestio-inline` корректно работает только при использовании **точечного импорта** (`. "github.com/aaa2ppp/contestio"`).
Обычный или алиасный импорт не поддерживается. Убедитесь, что ваш код использует именно точечный импорт, иначе утилита завершится с ошибкой.

## Производительность

Все бенчмарки проведены для **1 000 000 чисел** (`int` или `float64`) с буфером 4 КБ.  
**Система:** Intel Core i3-7100 @ 3.90GHz, Windows, Go 1.24.2.

*   **Время** указано в миллисекундах (ms) — меньше, значит быстрее.
*   **Память** указана в килобайтах (KB) — меньше, значит экономичнее.
*   **Аллокации** указаны в тысячах с суффиксом `k` (например, `1,046k`), единичные значения показаны как `1`.

**Ключевые выводы:**
*   Функции `contestio` (например, `scanSlice`) **в 6–9 раз быстрее** стандартного `fmt.Fscan`.
*   При использовании заранее выделенного слайса (или при выводе) **аллокации полностью отсутствуют**, что исключает нагрузку на сборщик мусора.
*   Внутри `contestio` используется та же техника с локальным буфером (`scratch`), приколоченным к писателю, что позволяет достичь максимальной производительности, предоставляя при этом удобный обобщённый интерфейс. Вам не нужно писать циклы с `AppendInt` вручную — библиотека делает это за вас, сохраняя эффективность ручного кода.

### Выбор парсера целых чисел (теги сборки)

По умолчанию используется базовая реализация (`base`) — она почти не уступает `fast` по скорости, но содержит меньше кода.  
При необходимости можно выбрать другую:

- `parse_int_fast` — максимальная производительность для десятичных целых.
- `parse_int_std` — использует `strconv.ParseInt`/`ParseUint` из стандартной библиотеки (максимальная надёжность).

Теги взаимоисключающие. Пример:  
`go build -tags=parse_int_fast`
